# Руководство по участию в разработке

Спасибо за интерес к проекту! Мы рады любому вкладу в развитие коллекции игр.

## Как начать

1. **Fork** репозитория
2. Создайте новую **ветку** для вашей функции: `git checkout -b feature/my-new-feature`
3. Внесите изменения и **commit**: `git commit -am 'Добавлена новая функция'`
4. **Push** в ветку: `git push origin feature/my-new-feature`
5. Создайте **Pull Request**

## Стиль кодирования

### C# Code Style

Следуйте стандартным соглашениям C#:

- Используйте **PascalCase** для имён классов, методов, свойств
- Используйте **camelCase** для локальных переменных и параметров
- Используйте **_camelCase** для приватных полей
- Добавляйте XML комментарии для публичных API

Пример:

```csharp
namespace GamesCore.Models;

/// <summary>
/// Описание класса
/// </summary>
public class MyClass
{
    private int _privateField;

    /// <summary>
    /// Описание свойства
    /// </summary>
    public string PublicProperty { get; set; }

    /// <summary>
    /// Описание метода
    /// </summary>
    /// <param name="parameter">Описание параметра</param>
    /// <returns>Описание возвращаемого значения</returns>
    public int MyMethod(string parameter)
    {
        var localVariable = 0;
        return localVariable;
    }
}
```

### XAML Style

- Используйте **PascalCase** для имён элементов
- Атрибуты размещайте по одному на строке для сложных элементов
- Используйте отступы 4 пробела

## Создание новой игры

### 1. Создайте класс игры

Наследуйтесь от `BaseGame`:

```csharp
using GamesCore.Models;

namespace YourNamespace;

public class YourGame : BaseGame
{
    public YourGame()
        : base("Название", "Описание", minPlayers: 2, maxPlayers: 4)
    {
    }

    public override void Initialize() { /* логика */ }
    public override void Start() { /* логика */ }
    public override void Pause() { /* логика */ }
    public override void End() { /* логика */ }
}
```

### 2. Добавьте тесты (планируется)

```csharp
[Test]
public void TestYourGame()
{
    var game = new YourGame();
    game.Initialize();
    // Проверки
}
```

### 3. Обновите документацию

Добавьте описание игры в соответствующие файлы документации.

## Добавление функций

### Новый интерфейс

Если вы добавляете новый интерфейс:

1. Поместите его в `src/GamesCore/Interfaces/`
2. Добавьте XML комментарии
3. Создайте базовую реализацию в `src/GamesCore/Models/`

### Новая модель

Если вы добавляете новую модель:

1. Поместите её в `src/GamesCore/Models/`
2. Реализуйте соответствующий интерфейс (если есть)
3. Добавьте XML комментарии

## Комментарии и документация

### XML комментарии

Обязательно добавляйте XML комментарии для:

- Публичных классов
- Публичных методов
- Публичных свойств
- Параметров методов

### Файлы документации

При добавлении значительных функций обновите:

- `README.md` - основная информация
- `docs/ARCHITECTURE.md` - архитектура
- `docs/GETTING_STARTED.md` - руководство по началу работы

## Pull Request Guidelines

### Что должен содержать PR:

1. **Описание** - что делает ваш код
2. **Причина** - зачем это нужно
3. **Тесты** - как проверить работу (когда тесты будут реализованы)
4. **Скриншоты** - для UI изменений

### Шаблон описания PR:

```markdown
## Описание
Краткое описание изменений

## Причина
Почему это нужно / какую проблему решает

## Изменения
- Изменение 1
- Изменение 2

## Тестирование
Как проверить работу изменений

## Скриншоты (если применимо)
[Прикрепите скриншоты]
```

## Сообщения коммитов

Используйте понятные сообщения коммитов на русском или английском:

- `Добавлена функция X` или `Add feature X`
- `Исправлена ошибка Y` или `Fix bug Y`
- `Обновлена документация Z` или `Update documentation Z`

### Примеры хороших коммитов:

```
✓ Добавлена игра "Угадай число"
✓ Исправлена ошибка при добавлении игрока в комнату
✓ Обновлена документация по архитектуре

✗ fix
✗ update
✗ changes
```

## Тестирование (когда будет реализовано)

Перед созданием PR убедитесь, что:

1. Код компилируется: `dotnet build`
2. Тесты проходят: `dotnet test`
3. Нет предупреждений компилятора
