# Архитектура проекта "Коллекция игр"

## Обзор

Проект представляет собой коллекцию игр с мультиплеером, включающую пошаговые настольные игры и интеллектуальные игры на основе телевизионных передач.

## Структура проекта

```
GamesCollection/
├── src/
│   ├── GamesCore/           # Основная библиотека с бизнес-логикой
│   │   ├── Interfaces/      # Интерфейсы для игр, игроков, комнат
│   │   ├── Models/          # Модели данных и базовые реализации
│   │   └── Services/        # Сервисы (будут добавлены позже)
│   │
│   └── GamesHub/            # WPF-приложение (лаунчер)
│       ├── App.xaml         # Точка входа приложения
│       └── MainWindow.xaml  # Главное окно
│
├── examples/                # Примеры использования
├── experiments/             # Экспериментальный код для тестирования
├── docs/                    # Документация
└── GamesCollection.sln      # Файл решения Visual Studio
```

## Компоненты системы

### 1. GamesCore (Ядро)

Основная библиотека, содержащая всю бизнес-логику игр.

#### Интерфейсы:

- **IGame** - Базовый интерфейс для всех игр
  - Содержит методы: Initialize(), Start(), Pause(), End()
  - Свойства: GameId, Name, Description, MinPlayers, MaxPlayers

- **IPlayer** - Интерфейс игрока
  - Свойства: PlayerId, Name, Role, IsConnected

- **IGameRoom** - Интерфейс игровой комнаты
  - Методы управления игроками: AddPlayer(), RemovePlayer()
  - Система чата: SendMessage()
  - Управление ролями: AssignRole()

#### Модели:

- **BaseGame** - Абстрактная базовая реализация игры
- **Player** - Реализация игрока
- **GameRoom** - Реализация игровой комнаты с мультиплеером

#### Перечисления:

- **PlayerRole** - Роли участников
  - Player (Игрок)
  - Moderator (Модератор)
  - Organizer (Организатор)
  - Host (Ведущий)
  - Operator (Оператор)
  - Editor (Редактор)
  - Spectator (Наблюдатель)

- **RoomStatus** - Статус комнаты
  - WaitingForPlayers (Ожидание игроков)
  - Ready (Готова к запуску)
  - InProgress (Игра идёт)
  - Paused (На паузе)
  - Finished (Завершена)
  - Closed (Закрыта)

### 2. GamesHub (WPF Лаунчер)

Графическое приложение для запуска и управления играми.

#### Функции:
- Главный хаб в стиле Microsoft Office
- Создание и управление игровыми комнатами
- Просмотр доступных игр
- Настройки приложения

## Система ролей

В игровых комнатах реализована система распределения ролей:

1. **Игрок (Player)** - Обычный участник игры
2. **Модератор (Moderator)** - Может управлять чатом и следить за порядком
3. **Организатор (Organizer)** - Создатель комнаты, имеет полные права
4. **Ведущий (Host)** - Для телевизионных игр
5. **Оператор (Operator)** - Технический специалист для телеигр
6. **Редактор (Editor)** - Управление контентом в телеиграх
7. **Наблюдатель (Spectator)** - Может только смотреть

## Мультиплеер

### Текущая реализация:
- Базовая структура для комнат с игроками
- Система чата внутри комнаты
- Управление подключением игроков

### Планируется:
- Сетевое взаимодействие через ASP.NET
- Синхронизация состояния игры
- Обработка отключений и переподключений

## Технологический стек

- **Язык**: C# (.NET 8.0)
- **UI Framework**: WPF (Windows Presentation Foundation)
- **Планируется**:
  - Unity (для более сложных игр)
  - ASP.NET (для серверной части мультиплеера)

## Этапы разработки

### Этап 1: Основа (Текущий)
- ✅ Базовая структура проекта
- ✅ Интерфейсы и модели игр
- ✅ Система ролей
- ✅ Базовый UI хаба

### Этап 2: Первая игра (Следующий)
- ⬜ Реализация конкретной игры
- ⬜ Тестирование игровой логики
- ⬜ Доработка UI для игрового процесса

### Этап 3: Мультиплеер
- ⬜ ASP.NET сервер
- ⬜ Сетевая синхронизация
- ⬜ Система чата с сервером

### Этап 4: Расширение
- ⬜ Добавление новых игр
- ⬜ Переход на Unity для сложных игр
- ⬜ Веб-версия хаба

## Принципы проектирования

1. **Модульность** - Каждая игра реализуется отдельно на основе общих интерфейсов
2. **Расширяемость** - Легко добавлять новые игры и функции
3. **Переиспользование** - Общая логика в GamesCore используется всеми играми
4. **Гибкость** - Поддержка различных типов игр и ролей

## Способы запуска (Планируется)

1. **Веб-версия** - Запуск хаба через браузер
2. **Desktop приложение** - Скачиваемое WPF приложение
3. **Отдельные игры** - Скачивание конкретной игры без хаба
